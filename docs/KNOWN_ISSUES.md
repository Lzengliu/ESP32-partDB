# V1.1 已知问题

[English Version](KNOWN_ISSUES_EN.md)

版本：V1.1

## 相机只有手动对焦

当前相机模组不支持 AF。二维码边缘清晰但仍出现 ECC/未识别时，应旋转镜头手动调整焦距，并保证二维码有足够尺寸、光照和静止时间。固件已移除无效的自动对焦入口；此问题不再作为 V1.1 发布阻塞项。

摄像头正常情况下会自动休眠并释放驱动。如果尚未点击“预览抓图”或“扫码”完成唤醒和预热，Web 总览/硬件检测显示“相机异常”属于预期状态，不表示模组损坏。发起预览或扫码后会按需重新初始化摄像头。

## Web 管理页没有访问认证

Web 页面及 `/api/*` 管理接口目前没有强制鉴权，其中包含配置修改、TF 格式化、资源删除和 OTA。V1.1 应只部署在可信局域网，不要把设备端口直接映射到公网。后续版本需要加入登录/设备密钥校验、CSRF 防护和危险操作权限分级。

## 外部字体未参与运行时渲染

Web 页面可上传、选择和删除 TTF/OTF/TTC/WOFF/WOFF2 文件，但 `ui_font_set_active_path()` 目前只保存路径。运行时使用固件内置 8/12/16 px 点阵字模，不会解析上传字体。

## 开机动态图未逐帧播放

静态 JPEG 开机图可以显示。GIF/WebP 开机动画可以上传和选择，但尚未集成逐帧解码器，启动阶段不会播放动画。

## 小屏长文本与多批次流程

320 x 480 屏幕会截断很长的简介、备注或参数。多批次对象在无法确定具体批次时会拒绝直接写库存，当前还没有独立批次选择页。

## NFC 依赖原型接线质量

PN532 已在当前原型上读取到固件版本并可启动服务，但 GPIO0 临时作为恢复复位脚，I2C 模式拨码、上拉、供电和线长仍会影响稳定性。正式 PCB 应提供独立、非启动绑带的复位脚，并按 `firmware/docs/wiring-and-bringup.md` 核对接线。

## 自动化测试仍有限

当前主要依赖严格编译警告、实机刷写、启动日志、HTTP 状态/OTA 回归和手动 UI 检查。尚缺 Part-DB mock、UI 状态机单测、扫码图像回归集和 NFC 硬件仿真。
