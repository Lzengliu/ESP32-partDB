# V1.0 遗留问题

版本：V1.0

## 1. NFC 仍可能离线

当前实机状态接口中 NFC 仍可能显示：

```text
nfc_ready: false
last_err: ESP_ERR_NOT_FOUND
```

已知现象：

- 手机靠近 PN532 可能触发手机卡包，但 ESP32 侧仍可能读不到 PN532。
- 信息页、快捷页、写 NFC 页可能出现状态理解不一致：读卡服务有轮询状态，但底层 PN532 仍未 ready。
- 当前硬件配置为 PN532 与 FT6336 共用硬件 I2C，SDA/SCL 为 GPIO35/GPIO36，PN532 地址 `0x24`，FT6336 地址 `0x38`。

需要继续排查：

- PN532 模块拨码/接口模式是否为 I2C。
- PN532 I2C 地址是否固定为 `0x24`，是否存在模块兼容差异。
- 共用 I2C 时 PN532 是否被触摸控制器、上拉电阻、线长或电平影响。
- 是否需要给 PN532 独立 I2C 或切回软件 I2C，并在 UI 中统一状态来源。

## 2. 外部字体上传未真正接入运行时渲染

V1.0 已修复 Web 字体上传的反馈问题，可以看到上传成功/失败结果，也能保存字体路径。

但当前固件运行时仍主要使用内置点阵字体：

- `ui_font_set_active_path()` 目前只记录选择路径。
- 未实现 TTF/OTF/WOFF 的解析和动态字形栅格化。
- 数学符号、单位符号等是通过内置符号字模补齐，不是通过上传字体即时渲染。

后续如果要完整支持外部字体，需要引入字体解析/栅格化方案，或者预生成点阵字库。

## 3. 详情资料展示仍受小屏空间限制

详情页 V1.0 已扩展资料显示区域，并拆成 3 页展示：

- 基础信息
- 简介/备注
- 参数信息

但 320 x 480 屏幕空间有限，长描述、长备注、长参数仍会被截断。后续可实现上下滚动、分页列表或长文本弹层。

## 4. 出入库批次处理仍偏保守

当前库存写回已接入 Part-DB，但多批次对象仍需要更明确的批次选择逻辑。

当前策略：

- 如果能从详情、条码、NFC 或搜索目标定位具体批次，则按该批次处理。
- 如果对象存在多个批次但没有具体批次 ID，UI 会提示需要扫码/NFC 具体批次。

后续应增加批次选择页或默认批次策略。

## 5. Web 管理页是内嵌字符串

`http_server.c` 中的 Web 页面目前以内嵌 HTML/JS/CSS 字符串实现。

优点：

- 单固件即可运行。
- 无需额外静态文件部署。

问题：

- 维护体验一般。
- 页面复杂度继续上升后不利于拆分和测试。

后续可把 Web 前端拆成独立资源构建流程，再嵌入或上传到文件系统。

## 6. 构建产物和源码仓库需要分离

本地工作区包含大量历史固件、硬件资料和第三方源码参考，不适合直接作为公开 GitHub 仓库。

V1.0 已准备 `.gitignore` 和独立打包目录，发布时应只上传整理后的源码包与 Release 固件包。

## 7. 自动化测试不足

当前主要验证方式是实机编译、刷写、HTTP 状态接口和手动 UI 测试。

缺少：

- Part-DB API mock 测试。
- UI 状态机单元测试。
- NFC/扫码流程模拟测试。
- Web API 回归测试。
