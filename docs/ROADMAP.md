# 可实现功能与后续路线

版本：V1.0

## 短期可实现

- NFC 状态统一：让信息页、快捷页、写入页全部使用同一底层 ready 状态和错误原因。
- NFC I2C 扫描/诊断页：显示总线扫描结果、地址 ACK、初始化阶段和最近错误。
- 详情页长文本滚动：简介、备注、参数支持上下滑动，而不是只截断显示。
- 批次选择页：多批次元器件出入库前显示可选批次。
- 搜索结果缓存：同一关键词重复进入时减少网络请求。
- Web API mock：提供本地测试脚本，验证 Part-DB 搜索、详情、库存写回路径。
- Release 自动校验：为固件包生成 SHA256SUMS，并在发布说明中列出刷写偏移。

## 中期可实现

- 外部字体真正渲染：支持预生成点阵字库或集成轻量字体栅格化。
- Web 前端拆分：将内嵌 HTML/JS/CSS 拆成可维护的前端源码，再构建进固件。
- UI 主题和布局配置：支持更多屏幕尺寸、横屏布局和显示密度。
- 库存操作日志：在 TF 卡记录最近出入库操作，便于离线追踪。
- 离线缓存：缓存常用元器件详情，网络断开时仍可查看基础资料。
- OTA 版本信息：固件内嵌语义化版本号、构建时间和 Git commit。

## 长期可实现

- 多 Part-DB 实例配置。
- 用户权限和操作审计。
- 标签打印机或外部扫码枪支持。
- 更完整的 NDEF 格式：支持 URI、JSON、Part-DB 深链接、批次信息。
- 硬件抽象层：支持不同屏幕、触摸、摄像头和 NFC 组合。
- 自动化硬件测试夹具：上电后自动验证屏幕、触摸、TF、NFC、摄像头。

## V1.0 后优先级建议

1. 先解决 NFC 底层连接和状态冲突。
2. 再做详情页长文本滚动和批次选择。
3. 最后处理外部字体真实渲染和 Web 前端拆分。

原因：NFC 和批次选择直接影响现场出入库准确性；字体和 Web 重构主要影响体验和维护效率。
