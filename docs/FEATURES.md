# 主要功能区说明

[English Version](FEATURES_EN.md)

版本：V1.1

## Part-DB

- 通过 Web 管理页保存 Part-DB base URL 和 API Token。
- 支持模糊搜索、结果缓存、详情、参数、库存和批次读取。
- 详情页可执行入库/出库并写回 Part-DB。
- 条码、二维码和 NFC 内容可路由到 Part、Lot、IPN 或条码查询。

## 设备端 UI

- 主页、搜索结果、详情、快捷操作、信息和设置页面。
- 详情资料分区展示，支持数量输入、入库、出库和写 NFC。
- 触摸软键盘、亮度设置、自动息屏和唤醒。
- 内置 8/12/16 px 中英文字模和常用符号。

## Web 管理

- 查看 WiFi、PSRAM、Part-DB、TF、显示、触摸、NFC、相机、扫码和 UI 状态。
- 保存网络、Part-DB、显示、触摸和资源配置。
- 管理 TF 文件、背景图、开机图、锁屏图和字体文件。
- 执行硬件诊断、相机预览、本地扫码和 OTA。
- 项目声明显示作者“灵异大队长”及开源地址。

## 相机与二维码

- PSRAM 帧缓冲与 SVGA 灰度扫码。
- 曝光稳定帧根据图像亮度自适应提前结束。
- ZXing-C++ 快速识别，quirc 回退，必要时再获取一帧重试。
- `/api/camera.jpg` 提供预览，`/api/camera/scan` 执行本地识别。

当前相机模组没有 AF，必须手动旋转镜头调焦；详见 `KNOWN_ISSUES.md`。

摄像头默认按需启用，操作结束后会自动休眠并释放驱动。因此，在尚未执行预览或扫码完成唤醒、预热时，Web 总览和硬件诊断显示“相机异常”属于正常现象。

## NFC

- PN532 使用独立硬件 I2C，不与触摸或相机 SCCB 共用总线。
- 后台轮询、总线恢复、NDEF 文本读取、写入和清除。
- NFC 与相机等操作通过外设仲裁避免并发冲突。

## TF 卡与资源

- FAT/FAT32 挂载、空间状态、目录初始化、上传、预览和删除。
- 静态开机图可在启动阶段解码显示。
- 外部字体文件可保存和选择；运行时仍使用内置点阵字模。

## OTA 与诊断

- 4.5 MiB factory/ota_0/ota_1 应用分区。
- OTA 上传前检查空包和分区容量。
- 启用 bootloader rollback，核心服务健康启动后确认新镜像。
- 状态接口提供固件版本、作者、仓库地址和硬件诊断信息。
