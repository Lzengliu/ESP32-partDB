# 主要功能区说明

版本：V1.0

## 1. Part-DB 连接

- 在 Web 管理页配置 Part-DB 地址和 API Token。
- 支持测试元件列表和分类接口。
- 设备端通过 Part-DB API 拉取元器件搜索结果、详情、参数和库存信息。
- 详情页出入库操作会写回 Part-DB 库存批次。

## 2. 设备端触摸 UI

- 顶部搜索框支持模糊搜索。
- 搜索结果列表可直接进入详情页，不再强制二次精准 IPN/条码验证。
- 底部导航包含主页、快捷、信息、设置。
- 点击底部导航会清空搜索框内容。
- 详情页展示元器件基础信息、简介、备注、参数、库位、条码、库存和批次信息。
- 详情页可输入出入库数量，并提供入库、出库、写 NFC 按钮。

## 3. 快捷操作

- 快捷页提供入库、出库、扫码、NFC、搜索入口。
- 入库/出库模式可提前选择，再通过搜索/扫码/NFC 进入对象。
- 当前终端侧不提供 Part-DB Web 的完整编辑工具，重点是现场查询和库存操作。

## 4. Web 管理页

- 查看 WiFi、Part-DB、TF 卡、显示、触摸、NFC、摄像头和 UI 状态。
- 保存设备名称、AP、显示、触摸、Part-DB、资源路径等配置。
- 上传字体、背景图、开机动画、锁屏背景。
- 查看和删除 TF 卡文件。
- 执行硬件诊断、屏幕测试、亮度设置、OTA 上传。

## 5. TF 卡和资源管理

- 支持 FAT/FAT32 TF 卡挂载。
- 自动准备缓存目录、图片目录、动画目录、字体目录。
- 支持 Web 上传资源文件并在设置页选择。
- 字体上传 V1.0 可保存路径并反馈结果，但运行时字体渲染仍以固件内置点阵字体为主。

## 6. 摄像头和二维码

- 使用 ESP32-S3 摄像头采集 JPEG。
- 使用 quirc 解码二维码。
- 扫码结果可进入 Part-DB 详情查询。
- Web API 支持触发拍照和扫码。

## 7. NFC

- 已接入 PN532 初始化、读卡服务、NDEF 文本写入流程。
- 写 NFC 在详情页通过确认弹窗启动，降低误触风险。
- NFC 当前仍是 V1.0 的重点遗留问题，具体见 `KNOWN_ISSUES.md`。

## 8. OTA 与维护

- Web 管理页支持 OTA 上传。
- 状态接口输出完整硬件和服务状态，便于远程排查。
- 支持硬件诊断页快速判断屏幕、触摸、NFC、TF 卡、摄像头状态。
